Tomcat WebSocket消息处理器

Quan 发表于 2017-07-08

简介

在WebSocket链接建立后进行消息传输时，会对消息进行编解码处理，然后才被消息处理器（javax.websocke.MessageHandler）处理。Api规范里面提供自定义编码（javax.websocket.Encoder），解码(javax.websocket.Decoder)以及消息处理器的接口，可用于自定义消息的处理。

在上面代码中可以看到，

  • 如果MessageHandler的参数类型是String, ByteBuffer, PongMessage, 框架会支持掉，不用编写Decoder;
  • 如果MessageHandler的参数类型是byte[],根据MessageHandler是否Whole子接口来判断使用wholeBinary类型消息处理器还是PartialBinary处理器,要编写Decoder实现javax.websocket.Decoder.Binary 或者java.websocket.Decoder.BinaryStream；
  • 如果MessageHandler的参数类型是InputStream 那就只能是WholeBinary的消息处理器，要编写Decoder实现javax.websocket.Decoder.Binary 或者java.websocket.Decoder.BinaryStream；
  • 如果MessageHandler的参数类型是Reader，那就只能是WholeText的消息处理器,要编写Decoder实现javax.websocket.Decoder.Text或者java.websocket.Decoder.TextStream；
  • 如果是其他类型，所有的decoders中找到匹配的，需要编写对于的decoder。

编码encoder

当向外发送消息时，需要进行编码，规则如下：

  • 如果没有找到Encoder，但是是原生数据类型，编码成字符串处理；
  • 如果没有找到Encoder，但是是byte[]，编码成ByteBuffer处理；
  • 如果找到Encoder, 则调用相应的decode method处理。
